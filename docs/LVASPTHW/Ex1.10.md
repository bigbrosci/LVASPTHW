##分享一下我的工作窗口


首先说一下我自己对计算过程的理解。在AI出来之前，本质上来说我们需要面对的东西有2个：本地电脑和服务器。

在本地电脑我们可以生成计算的模型，输入文件。在服务器上运行任务。它们之间的功能也是有一些overlap的，比如输入文件的生成，结果的分析等，通过本地电脑和服务器都可以进行。只是不同人的习惯罢了。但简而言之我们需要本地电脑和服务器。 

在这个基础上，可以延伸出很多的细节，输入文件的创建，文件的传输，模型可视化搭建，输出文件的分析，数据可视化，画图等等。我们需要太多的软件来实现这些零零碎碎的功能。根据操作系统的不同，用户习惯的不同，这些复杂性又进一步的增加。

在我看来，最简单的莫过于通过`terminal` 来管理控制不同的软件，工作流。一方面terminal 可以实现本地电脑和服务器的连接，可以编辑输入文件，也可以通过命令来调用不同的可视化软件，运行bash或者python命令生成输入文件以及分析结果，画图等等，写文章可以在terminal中通过vim+latex的形式来进行。这也是本人所最爱的工作方式：尽可能的把不同的细节流程整合到`terminal`中。只要有terminal，我就可以干活。但这也不代表我不会使用其他的软件，Excel, PPT之类的，该用还得用，有的服务器只能网页登录，我也老老实实的按照人家的方式进行。大部分的工作主要还是通过`terminal`来进行的。

当AI重塑我们的生活工作方式之后，由于需要跟AI交互，去优化我们的工作细节，提高工作效率。我们就需要给AI一个发挥的空间。常用的方式就是在网页上或者不同公司推出的程序进行交互。比如写个脚本，先让AI写好，然后自己复制或者下载到本地去运行。这样存在的一个弊端就是需要在之前的工作方式和AI之间来回往返。比如，AI给了一个有bug的脚本， 你要返回给AI，让它修改， 运行，再返回。。当重复的次数多了的时候，AI也会存在降智的问题，我们的耐心也会被消磨殆尽。 在体验了不同的工作方式之后，总结了一下当前最适合本人的工作窗口，希望对大家有所帮助。这个窗口，其实就是`Visual studio code` 的界面。 

下面主要是从以下7个方面逐一去简单介绍下。

([工作窗口截图](figs/ex1.10_1.png))



1） 文件目录，通过`File` 去定位到你的文件夹。

2） Github的登录按钮，如果你购买了GitHub的copilot，就需要在这里先登录账号，然后才可以使用不同的大模型。

3） 展现脚本的内容，当你双击1）中的脚本时，就会出现，你可以自己修改，也可也通过AI来修改（见5）。

4） `Terminal` ，在这里继承了之前terminal可以做的所有的事情。

5）AI-交互区：在这里，你可以选择不同的大模型语言去帮你写脚本，分析结果等。值得逐一的是，当你在1）中打开一个目录或者在3）中打开一个脚本的时候，它会自动定位到当前的目录和脚本。 

6）当你向AI提问之后，它会啰嗦一阵子，记录以下思考，行动的的过程，这也显示在区域6中。

7）这里额外提一下，如果你买了ChatGpt，可以在VS Code 中使用Codex。如果你没有ChatGpt， 则使用的是Github的`Chat`这个按钮（GitHub Copilot ）。自己可以来回切换，不过这需要自己安装插件，去问下AI就可以解决：“如何在VS Code界面上安装GitHub Copilot 插件？或者如何安装OpenAI插件。**Codex vs Github Copilot 的选择**：这取决于你的财力以及对大模型的需求。简单来说，如果你已经购买了ChatGpt，也就自动有了Codex的使用权。如果Codex满足不了你的话，就可以选择Copilot，可以在Gemini，Claude，等模型中自由切换。本人现在购买的是ChatGpt，因为除了写代码还有写文章之类的也用得着。

**小结**：计算流程的复杂性，不同的操作系统，软件，再结合人们不同的操作喜好，这也导致了工作方式的万千变化。总而言之就是在自己舒服的一个窗口去实现工作效率的最大化，让自己开心愉快。这也需要自己去尝试不同的方式，然后磨合。希望我的这个工作方式也能对你有所启发。如果你有不同的工作方式，可以写一写分享给大家，但是请不要去尝试反驳我，因为这玩意没有固定答案，适合自己的才是最好的。PS：本文纯手写，未经AI润色，可能有错别字，请见谅。
